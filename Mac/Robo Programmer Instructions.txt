Robo Programmer - User Instructions
====================================

Purpose
-------
Robo Programmer is a local browser-based editor for working with FANUC LS files. It provides project management, guided TP instruction builders, diagnostics, Data Assignments editing, project-wide reference tools, and portable project packages.

The tool runs locally on your PC. Project files are not uploaded to an online service.


Launching The Tool
------------------
Extract the deployment zip first. Do not run the tool from inside the compressed zip.

Use either launcher in the extracted main folder:

  Robo Programmer V5.0.cmd
  Launch Robo Programmer.cmd

The launcher starts a visible local PowerShell server window and opens Robo Programmer in your default browser. Keep the PowerShell server window open while using the tool. Python, Node, and administrator rights are not required.


Optional Desktop Shortcut
-------------------------
After extracting the deployment zip, double-click:

  Create Desktop Shortcut.cmd

This creates a branded Robo Programmer icon on your Desktop without installing anything or requiring administrator rights.

The shortcut points back to the extracted deployment folder. Keep that folder in the same location after creating the shortcut.

When you receive a newer deployment:

  1. Extract the new deployment.
  2. Run its Create Desktop Shortcut.cmd.
  3. The existing Robo Programmer Desktop shortcut is replaced automatically and points to the new deployment.


Start Screen And Recent Projects
--------------------------------
The start screen provides three ways to begin:

Open Project Folder:
  Opens an existing normal project folder.

Open ROBO Package:
  Opens a portable .ROBO project package, validates it, asks where to unpack it, creates a new project folder, and opens the unpacked project.

Create New Project:
  Creates a new project folder and starter files.

Recent Projects:
  Shows recently opened project folders for quick reopening. Folder permission may need to be approved again after restarting the browser.

Clear:
  Clears the Recent Projects list. This does not delete any project folders.


Project Structure
-----------------
When you create a new project, the tool creates a normal Windows folder containing:

  LS Files
    Trash
  Spreadsheet
    Data Assignments.xlsx
  History
  Robot Backups
  Project manifest
  Project launcher

LS Files:
  Contains the active robot .LS programs.

LS Files\Trash:
  Stores timestamped copies of LS files deleted through Robo Programmer.

Spreadsheet\Data Assignments.xlsx:
  Stores descriptions for registers, I/O, position registers, frames, timers, and other supported references.

History:
  Reserved for future history and version features.

Robot Backups:
  Stores timestamped ZIP files created through the Robot Backup workspace.

Project launcher:
  Opens Robo Programmer and reconnects to that project.

Project configuration updates:
  When an older project folder or .ROBO package is opened, Robo Programmer automatically updates its project manifest, launcher, and missing support folders/files to the current project configuration. Existing LS files, Data Assignments.xlsx content, favorites, and custom manifest fields are preserved.


Go Online
---------
The compact Go Online control is in the top bar next to the Robo Programmer title.

At startup, Go Online is gray. Choose it to connect Robo Programmer to the selected robot. If no robot address has been saved yet, Robo Programmer prompts for the robot IP address or host name.

Use the small drop-down next to the button to select a previously used robot address or add another robot address.

When the robot connects successfully, the button turns green and shows a blinking green indicator. Robo Programmer refreshes the online check every few seconds. If the connection fails or is lost, the button reports Connection Failed. While online, choose the button again to go offline.

The saved robot address is reused by Robot Comment Push, Robot Export, Robot Backup, and Live Position. When a project is open, the address is also saved into that project's config.

When Go Online succeeds while a project is open, Robo Programmer automatically runs the Robot Export Check Connection workflow so the Robot Export page already knows whether FTP Interface (R796) and ASCII Program Loader (J716) are available.


File Menu
---------
On the startup screen, the File menu shows Open Project, New Project, and Open ROBO Package. After a project is opened, those startup actions are hidden and the project actions are shown.

Open Project:
  Selects and opens an existing project folder.

New Project:
  Asks for a parent location and project name, then creates the standard project structure and a starter MAIN.LS file.

Open ROBO Package:
  Imports and unpacks a portable .ROBO project.

Export Project as ROBO:
  Packages the full open project into one portable .ROBO file. The package includes the project folder structure, spreadsheets, and current unsaved LS editor changes.

Save:
  Shows a confirmation warning, then saves all pending changes in every modified LS file back to the project folder.

Sync Data Assignments:
  Applies spreadsheet descriptions to LS references and backfills blank spreadsheet descriptions from LS files.

Robot Comments (Beta):
  Opens the guarded robot-comment comparison and upload tool in the Data Assignments workspace. See Robot Comments Beta below before using it.

Close Project:
  Closes the project and returns to the start screen. The tool warns before clearing unsaved working-copy changes.

Theme:
  Changes the app theme between Light, Gray, and Dark.


ROBO Project Packages
---------------------
A .ROBO file is a portable package containing a complete Robo Programmer project.

To send or archive a project:

  1. Open the project.
  2. Choose File > Export Project as ROBO.
  3. Select where to save the .ROBO file.

To open a package:

  1. Choose Open ROBO Package from the start screen or File menu.
  2. Select the .ROBO file.
  3. Choose the parent folder where the project should be unpacked.
  4. Robo Programmer creates a uniquely named project folder and opens it normally.

ROBO packages are validated before extraction. Existing project folders are not overwritten during import.


Working With LS Files
---------------------
The Project panel on the left shows all active .LS files.

Click an LS file:
  Opens it in the main editor.

Favorite star:
  Select the star beside an LS file to add it to project favorites. Favorite files appear alphabetically at the top of the list. Select the filled star again to remove the favorite. Favorites are stored with the project and do not modify LS file contents.

New LS:
  Creates a new uppercase LS file from the starter template and inserts it alphabetically.

Save Current LS:
  Saves pending changes in whichever editor is active. The button is enabled when that active file has unsaved changes.

Delete LS:
  Asks for confirmation, copies the selected LS file into LS Files\Trash with a timestamped name, and removes it from the active project.

Refresh:
  Reloads the LS file list from disk. If a file has unsaved changes, choose Save, Discard, or Cancel Refresh.

Dirty-file indicators:
  Modified files are marked in the file list and editor. Pending changed lines are also marked in the editor until saved.


Editor Basics
-------------
The editor hides the LS header by default so you can focus on the /MN instructions.

Show Header:
  Reveals or hides LS data above the /MN section.

Show Footer:
  Appears when data exists after /END and reveals or hides that footer data.

Split:
  Opens a second editor beside the main editor. Select another LS file from the Split View dropdown. Teach Pendant and Advanced TP actions apply to whichever editor was used most recently.

Copy Path:
  Copies the full Windows path of the current LS file to the clipboard when the project was opened from its generated launcher.

Export:
  Exports the currently open LS file to another location without saving over the original project file.

Undo / Redo:
  Uses the toolbar buttons or normal Ctrl+Z and Ctrl+Y keyboard shortcuts in the main editor.

A- / A+:
  Changes the editor font size.

Cursor status:
  Displays the current line and column.

Resizable panels:
  Drag the vertical dividers to resize the Project panel and the right-side tools.


Editing Behavior And Line Numbering
-----------------------------------
Robo Programmer manages visible /MN instruction numbering while you edit.

Press Enter:
  Creates a new numbered instruction line containing a blank terminator:

     ;

Press Backspace on a raw blank numbered line:
  Deletes the full blank line in one press.

Press Tab:
  Inserts two spaces.

When numbered lines are added, removed, moved, or edited, visible TP instruction numbers are renumbered.


Autocomplete Suggestions
------------------------
Typing supported LS commands or sections opens an autocomplete suggestion list near the cursor.

Arrow Down / Arrow Up:
  Changes the active suggestion.

Enter:
  Applies the active suggestion.

Escape:
  Closes suggestions.

Mouse click:
  Applies the selected suggestion.

The suggestion list automatically opens above or below the active line based on available screen space.


Comments, Remarks, And Tag Display
----------------------------------
Comment:
  Inserts a FANUC comment line beginning with !.

Remark:
  Toggles // in front of the logic on the current numbered line or every selected numbered line.

FANUC ! comments are limited to 32 characters. Longer comments automatically wrap into additional comment lines.

Toggle Comments:
  Shows or hides Data Assignments descriptions on supported references in the active LS file. Before hiding comments, blank spreadsheet descriptions can be backfilled from names found in that LS file. This is a view-only toggle and does not mark the LS file or its lines as changed.

Toggle State:
  Shows or hides robot-export state values such as ON, OFF, or numeric values while preserving reference descriptions. This is a view-only toggle and does not mark the LS file or its lines as changed.


Teach Pendant Menu
------------------
The Teach Pendant panel provides guided builders and prompts for common TP instructions. New instructions are inserted above the active editor cursor.

Motion Instruction:
  Builds J, L, or C motion using P or PR targets, speed units, FINE, and CNT terminations.

WAIT Instruction:
  Builds DI state, R comparison, or timed WAIT instructions.

I/O Instruction:
  Builds output assignments or input condition actions for DO, RO, GO, AO, DI, RI, GI, and AI.

Additional tools:

  R Register
  IF / THEN
  LBL
  JMP LBL
  CALL
  Comment
  Remark
  Toggle Comments
  Toggle State
  Frame / Tool
  Timer

Each guided builder shows a preview before insertion.


Advanced TP
-----------
Advanced TP provides broader FANUC-style INST and EDCMD actions. Choose INST or EDCMD, select a category, then choose a command.

INST categories and examples:

Motion:
  J/L/C Motion, Offset Motion, Tool Offset, Skip Condition

Registers:
  R Assign, R Math, PR Assign, PR Element

I/O and Wait:
  Digital Out, Value Out, Pulse Output, WAIT DI/RI, WAIT GI/R, WAIT Time

Control:
  IF Action, SELECT, LBL, JMP LBL, CALL/RUN, FOR Block

Frames:
  UFRAME/UTOOL, Payload

Misc:
  Timer, Message, UALM, Comment, PAUSE

EDCMD categories and examples:

Line Edit:
  Insert Blank Above/Below, Duplicate Line, Delete Line, Clear Line, Move Line Up/Down

Comments:
  Remark Toggle, Comment Above

Search:
  Find, Replace

Format:
  Renumber


Project Tools
-------------
Open Project Tools on the right side to search and safely change references across every LS file in the open project.

Reference Search:
  Enter an exact supported reference such as R[43], PR[19], F[3], LBL[22], or P[1].

Search results show:

  File name
  Source line
  Reference category such as read, write, definition, jump, motion, or comment
  Matching line text

Click a result to open its LS file and jump to the matching line.

Safe Reassignment:
  Changes one exact reference number to another matching reference type across the full project.

Before applying a reassignment, Robo Programmer:

  Confirms the source is used.
  Blocks reassignment when the target is already used.
  Blocks incompatible reference types.
  Checks the Data Assignments target for conflicts.
  Shows a project-wide preview.
  Moves the Data Assignments description when appropriate.

Reassigned LS files remain marked as modified until reviewed and saved.


Diagnostics And Program Map
---------------------------
Open Diagnostics and click Check to run first-pass checks on the current main-editor LS file.

Checks include common issues such as:

  Missing required LS sections or program name
  Malformed, duplicate, or out-of-order TP lines
  Missing semicolons
  Missing motion terminations
  Unexpected or unclosed ENDIF blocks
  Duplicate labels
  Missing P position definitions

Diagnostic actions:

Go To:
  Opens and highlights the issue line.

Review Fix:
  Shows a before-and-after preview for supported automatic fixes. Choose Apply Fix only after reviewing the change.

Dismiss:
  Hides that diagnostic until Check is pressed again.

Program Map:
  Summarizes the program name, section count, /MN line count, label count, and position count.


Data Assignments Workspace
--------------------------
After opening a project, use the Data Assignments tab at the top to view and edit Spreadsheet\Data Assignments.xlsx directly in Robo Programmer.

The workspace organizes supported assignment types into spreadsheet tabs, including:

  DI / DO
  AI / AO
  GI / GO
  RI / RO
  UI / UO
  SI / SO
  Flags
  Numeric Registers
  UF / TF
  PR
  String Registers
  Timers

Edit a description:
  Change a description field. The updated description is saved into Data Assignments.xlsx.

Reload Spreadsheet:
  Reloads the workbook from disk and discards the currently displayed workbook copy. Each assignment section heading refreshes to show the actual number of available items found in the spreadsheet.

Push LS Descriptions to Data Assignments:
  Writes visible description edits and descriptions found in project LS files into Data Assignments.xlsx.

Sync To LS Files:
  Applies spreadsheet descriptions to matching references in the project LS files and backfills blank spreadsheet descriptions from LS files.

Description text is cleaned and limited to the supported FANUC name length.


Robot Comments Beta
-------------------
Robot Comments Beta compares descriptions in Data Assignments.xlsx with comments currently stored on a connected FANUC robot or RoboGuide virtual robot. It can then push only the explicitly selected comment changes.

This is a beta implementation. Begin testing with a backed-up RoboGuide virtual robot before connecting to a physical robot. Always review the proposed changes before pushing them.

Open the tool:

  1. Open a project.
  2. Select the Data Assignments workspace.
  3. Expand Robot Comments Beta below the spreadsheet tabs.

Robot IP address or hostname:
  Enter only the robot address, without a page path or credentials. Examples:

    192.168.1.10
    127.0.0.1:9000

Remember for this project:
  Stores the entered robot address in the project configuration. The address is restored when that project is reopened. Passwords or credentials are not stored.

Connect & Compare:
  Connects to the robot comment interface, reads supported robot comments, and compares them with the currently loaded Data Assignments.xlsx descriptions. This comparison step does not change the robot.

The comparison table shows:

  Push:
    Selects whether that proposed change will be sent.

  Assignment:
    Shows the assignment type and number, such as R[200] or DI[1].

  Spreadsheet:
    Shows the description currently loaded from Data Assignments.xlsx.

  Robot:
    Shows the comment currently stored on the connected robot.

  Status:
    Shows Change, Matches, Skipped blank, or Too long.

Push Selected Comments:
  Shows a final confirmation warning, then sends only checked rows marked Change. Spaces and supported punctuation are encoded in the format expected by the FANUC comment interface. After sending, Robo Programmer waits for the controller, reads the comments again, and retries verification before reporting any remaining mismatch with its expected and actual text.

Robot Comments Beta safety behavior:

  - Only comments are changed. Register values and I/O values are never changed.
  - Blank spreadsheet descriptions are skipped and do not clear existing robot comments.
  - Comments that exceed the robot's supported limit are blocked instead of silently shortened.
  - Matching comments are not sent.
  - Unsupported Data Assignments types are excluded.
  - Every push requires selected rows and a final confirmation.
  - Comments are read back from the robot after pushing for verification.

Supported robot comment types and limits:

  Numeric Registers, R: 16 characters
  Position Registers, PR: 16 characters
  String Registers, SR: 16 characters
  Robot I/O, RI and RO: 24 characters
  Digital I/O, DI and DO: 24 characters
  Group I/O, GI and GO: 24 characters
  Analog I/O, AI and AO: 24 characters
  Flags, F: 24 characters

The current robot comment interface does not expose UI/UO, SI/SO, UF/TF, or Timer comments, so Robo Programmer does not attempt to send those types.

Recommended beta test:

  1. Back up the RoboGuide virtual robot project.
  2. Put a recognizable description into one disposable Data Assignments row.
  3. Open Robot Comments Beta and enter the RoboGuide address, such as 127.0.0.1:9000.
  4. Choose Connect & Compare.
  5. Confirm the spreadsheet and robot values shown for that row.
  6. Uncheck every other proposed change.
  7. Choose Push Selected Comments and review the final confirmation.
  8. Verify the success message and confirm the updated comment in the robot's browser comment tool.

If connection or verification fails, do not repeatedly push changes. Confirm the robot address, network access, robot web server availability, and supported controller software before retrying.


Robot Export
------------
Open a project and choose Robot Export from the workspace buttons at the top of Robo Programmer.

Robot Export sends selected LS programs to a live robot using the robot's FTP server. The FANUC ASCII Program Loader option compiles each accepted LS file into a TP program. The same program list can also import selected robot programs into the current project's LS Files folder.

Required robot options:

  FTP Interface (R796)
  ASCII Program Loader (J716)

ASCII Upload is not required for Robot Export.

The Export LS Programs To A Robot instructions panel can be collapsed using its title bar. Collapsing it gives the program list more vertical space. Robo Programmer remembers this preference in the browser.

Robot Export workflow:

  1. Enter the robot IP address or hostname. The same address used by Robot Comments can be used.
  2. Choose Check Connection. Robo Programmer separately checks FTP Interface and ASCII Program Loader, displays whether each requirement is available, and only enables exporting when both checks pass.
  3. Review the program list. Existing LS or TP programs are marked as overwrite risks. Robot programs that are available online but missing from the current project are marked as available on the robot but not in the project.
  4. To replace an existing robot program, enable Allow overwriting existing robot programs. Existing programs remain locked and cannot be selected until this option is enabled.
  5. Select only the LS programs that should be exported or replaced.
  6. Choose Export Selected Programs and carefully review the final confirmation. Programs that will be replaced are listed separately.
  7. Wait for verification. Robo Programmer checks that the LS file and compiled TP program appear on the robot.
  8. Review and test the resulting TP program safely before production use.

Robot Import workflow:

  1. Go Online and choose Check Connection.
  2. Select project programs that should be replaced from the robot, or robot-only programs that should be added to the project.
  3. Choose Import Selected Programs and review the confirmation.
  4. Existing project LS files are archived in LS Files\Trash before replacement.
  5. Robot-only programs are downloaded as LS files and added to the project's LS Files folder.
  6. Imported files appear in the LS Editor project file list.

Robot Export safety behavior:

  - Every export requires an explicit selection and final confirmation.
  - Existing LS or TP programs are clearly marked and locked by default.
  - Existing programs can only be selected after Allow overwriting existing robot programs is explicitly enabled.
  - Existing programs are not overwritten unless the replacement warning is confirmed.
  - Replacement authorization is attached to each confirmed existing program. Other existing programs remain blocked even during the same export request.
  - Invalid or missing /PROG names are blocked.
  - Current unsaved editor contents are included, and the confirmation identifies those files.
  - Robo Programmer verifies the uploaded LS file and compiled TP program after export.

Important:

  Never export a program while it is running or selected for execution. Robot Export cannot reliably determine whether a target program is currently active on every controller configuration. Back up important robot programs and follow the site's robot safety procedures before overwriting programs.

When replacing an existing program, the robot must allow the ASCII Program Loader to rebuild its TP program. If the program is running, selected for execution, or protected, the robot may return only ASBN-008 and refuse replacement. Robo Programmer translates that response into an actionable message and reports that the existing program was preserved.

If an LS file cannot be compiled, Robot Export reports the robot's latest ASBN ASCII Program Loader error, including the reported line and column when available. Correct the LS source and export it again. Do not run a program unless Robot Export verifies its TP program.

Some controllers export valid LS programs without an /APPL header, while other controllers require that header before they will compile the same program. When needed, Robot Export adds a standard /APPL compatibility header to the transmitted copy and reports that adjustment in the export result. The LS file in the Robo Programmer project is not changed.

Programs created by older Robo Programmer templates may use a minimal LS header that newer controllers reject at /END. Robot Export detects that older header and rebuilds it in the controller-compatible format before transmission. Program instructions and the project LS file remain unchanged.

After updating Robo Programmer, close any older Robo Programmer server window and launch the latest version again. The launcher automatically chooses a fresh local port when an older incompatible server is still running.


Robot Backup
------------
Open a saved project and choose Robot Backup from the workspace buttons at the top of Robo Programmer.

Robot Backup reads files exposed by the robot's FTP Interface (R796) and saves a timestamped ZIP inside the project's Robot Backups folder. The operation reads robot files and does not change the robot.

Available backup types:

  ASCII Programs (.LS)
  TP Programs (.TP)
  All Available Robot Files

Robot Backup workflow:

  1. Choose a backup type and enter the robot IP address or hostname.
  2. Choose Inventory Robot Files.
  3. Every matching file is selected by default. Review the list and use individual checkboxes, Select All, or Clear Selection as needed.
  4. Sort the list alphabetically by file name or file type as needed.
  5. Choose Back Up Selected Files and confirm the operation.
  6. Watch the live progress bar for the current filename and completed-file count.
  7. Wait for the ZIP to be written into the project's Robot Backups folder.
  8. Review robot-backup-manifest.json inside the ZIP for the requested files, successful downloads, failed downloads, and file sizes.

All Available Robot Files means every selected file exposed through the robot's FTP server. It is not a FANUC controller image backup and may not be identical to the teach pendant's All of Above backup. Files that the controller does not expose or refuses to download cannot be included.

Closing or reloading the browser window while a robot backup is running triggers a warning because the partial backup has not yet been saved.

When an older project is opened, Robo Programmer automatically adds the Robot Backups folder if it is missing.


Live Robot Position
-------------------
Open a saved project and choose Live Position from the workspace buttons at the top of Robo Programmer.

Live Position reads the robot controller's current position page and displays:

  Joint position values for the active motion group.
  Cartesian position in the active user frame, including UF number, tool number, and configuration.
  Cartesian world position, when reported by the controller.
  Controller date/time, robot identifier, connection address, and last update status.

Live Position workflow:

  1. Enter the robot IP address or host name. The same saved robot address used by Robot Export, Robot Backup, and Robot Comment Push is reused when available.
  2. Choose Read Once for a single position snapshot.
  3. Select a refresh interval and choose Start Live Updates to poll the robot continuously.
  4. Choose Stop when live updates are no longer needed.

Live Position is read-only. It reads /MD/CURPOS.DG from the robot web server and does not move the robot, load programs, change comments, or write files.

If the display stops updating, confirm that the robot web page is reachable in the browser and that the address includes the correct port when using RoboGuide, such as 127.0.0.1:9000.


Saving And Session Behavior
---------------------------
Robo Programmer keeps a browser working copy while you edit and restores recent session information when possible. Large recovery snapshots are stored in the browser's IndexedDB storage so normal large projects are not limited by the much smaller localStorage quota.

Changes are not fully committed to the project folder until the affected LS files are saved.

Use one of these methods:

  Save Current LS for the active editor file.
  File > Save to confirm and save all pending LS file changes.
  Save when prompted during Refresh.

After project-wide reassignment or Data Assignments sync, review and save all modified LS files.


Recommended Workflow
--------------------
1. Extract and launch Robo Programmer.
2. Optionally create the Desktop shortcut.
3. Open a project folder, create a project, or import a .ROBO package.
4. Select an LS file from the Project panel.
5. Edit manually or use Teach Pendant and Advanced TP tools.
6. Use Project Tools when searching or reassigning references.
7. Run Diagnostics and review suggested fixes.
8. Use Data Assignments when descriptions must be edited or synchronized.
9. Save all modified LS files.
10. Export a .ROBO package when sending or archiving the complete project.


Important Notes
---------------
Robo Programmer edits LS text files. Always review changes before loading files into a robot or RoboGuide.

Suggested diagnostic fixes and project-wide reassignments include previews, but they must still be reviewed by a programmer.

Keep backups of important robot projects. Exporting a .ROBO package is a convenient way to create a portable project snapshot.

Do not move or delete an extracted Robo Programmer deployment folder after creating a Desktop shortcut unless you create a new shortcut from the new location.
