Robo Programmer V5.0 - Deployment Notes
=========================================

Goal
----
Create a zip that a normal Windows user can extract and launch without Codex, Python, Node, or a development setup.

Confirmed Working Process
-------------------------
Use the V1.0-style TcpListener deployment.

The confirmed working tester zip was built with:

  Package Deployable V5.0.ps1

That script validates the required files and creates a zip in:

  C:\Users\criedelsheimer\Documents\Fanuc Programmer Project\Test Exports\V5.0

The tester should:

  1. Extract the zip.
  2. Open the extracted folder.
  3. Optionally double-click Create Desktop Shortcut.cmd.
  4. Double-click Robo Programmer V5.0.cmd or the new Desktop shortcut.
  5. Keep the PowerShell server window open while using the tool.

Important Files
---------------
Launch Robo Programmer.cmd
  Main tester entry point. This must use relative paths based on %~dp0.

Create Desktop Shortcut.cmd
  Optional no-install helper. Creates a branded Desktop shortcut that points back to
  Launch Robo Programmer.cmd in the extracted portable folder.
  Running it from a newer deployment replaces the existing Robo Programmer.lnk shortcut.

start-server.ps1
  Older portable-server attempt. Do not use as the primary tester launcher.

Start-RoboProgrammerServer.ps1
  Primary portable local web server. This uses built-in Windows PowerShell and TcpListener.
  This matches the V1.0 deployment approach that worked for other users.

app.js
  Project launchers generated by the app must use the appRoot query parameter/localStorage value.
  Do not hardcode C. Riedelsheimer's development folder into generated project launchers.

README.md and Robo Programmer Instructions.txt
  Tell testers to extract the zip first, then double-click Launch Robo Programmer.cmd.

How To Build The Tester Zip
---------------------------
From PowerShell:

  powershell -NoProfile -ExecutionPolicy Bypass -File ".\Package Deployable V5.0.ps1"

Default output folder:

  C:\Users\criedelsheimer\Documents\Fanuc Programmer Project\Test Exports\V5.0

Validation Checklist
--------------------
1. app.js passes node --check when the bundled Codex Node runtime exists.
2. The zip contains:
   - index.html
   - styles.css
   - app.js
   - Data Assignments.xlsx
   - Create Desktop Shortcut.cmd
   - Launch Robo Programmer.cmd
   - Robo Programmer V5.0.cmd
   - Start-RoboProgrammerServer.ps1
3. Tester extracts the zip before launching.
4. Tester opens Launch Robo Programmer.cmd, not index.html.
5. Browser opens http://127.0.0.1:4220/.

Known Packaging Lessons
-----------------------
Do not send a raw source zip that depends on Python. Testers may not have Python installed, and their machines will not have the Codex bundled runtime path.

Do not use HttpListener as the primary V2 deployment server. It worked on the development PC but may fail on tester PCs due to Windows URL reservation or policy behavior. Use the V1.0-style TcpListener server instead.
